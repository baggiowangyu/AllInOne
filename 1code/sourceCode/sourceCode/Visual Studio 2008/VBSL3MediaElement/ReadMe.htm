Work with MediaElement

 

Introduction

 

MediaElement is an essential control to create a media player in Silverlight. It supports playing, pausing, stopping audio/video as well as a lot of interesting features. You can simply control MediaElement by its properties, events and methods.

The following points are covered in this sample:

·         How to get the total duration of audio/video

·         How to create a progress bar for MediaElement

·         Volume control

·         Play/Pause/Stop audio/video

·         Events of MediaElement (MarkerReached, MediaEnded, MediaOpened, MediaFailed, CurrentStateChanged)

·         How to handle MediaElement in full-screen mode

How to get the total duration of Audio/Video

 

You can use the following properties of MediaElement to get the total duration of audio/video that MediaElement plays:

MediaElement.NaturalDuration.TimeSpan

One note is, you can only get the total duration after MediaOpended event.

 

How to create a progress bar for MediaElement

 

We can use Slider control to act as the progress bar of our player. In this sample, the idea to do this is as below.

·         Initialize Slider.Maximum in MediaOpended event.

·         Use a DispatcherTimer to syncronize the current position of the video with the Value properties of Slider control

·         Customize the style of Silder control, in which way we can catch DragStarted/DragCompleted event of the Thumb control. We can then change the position of MediaElement in these two event handlers.

 

Volume control

 

In this sample we use another Slider control to control the volume of MediaElement. Only one line of code is needed to achieve this:

this.MyMediaElement.Volume=this.VolumeSlider.Value;

The above code is put in the ValueChanged event handler of Slider control.

 

Play/Pause/Stop audio/video

 

We can play/pause/stop audio/video by calling MediaElement.Play()/MediaElement.Pause()/MediaElement.Stop() methods

 

Events of MediaElement

 

MarkerReached

This event fires when a marker of the MediaElement is reached.

After markers are loaded from the video and assigned to the Markers properties of MediaElement, we can add our own markers. If we want to do so, a proper place is MediaOpened event.

 

Code to add markers for MediaElement:

this.MyMediaElement.Markers.Add(new TimelineMarker()

          {

              Text = "This marker is added by code!",

              Time = TimeSpan.FromSeconds(8),

              Type = "MyMarker"

          });

To learn how to add markers for a video in Expression Encoder, please refer to:

http://expression.microsoft.com/en-us/cc482957.aspx

 

MediaEnded

This event fires when the media has ended. Common usage of this event is to reset state of certain controls or replay audio/video.

 

MediaOpened

This event fires when the media stream has been validated and opened, and the file headers have been read.

As described in MSDN, for this event we must take care of following things:

Properties in this second category will not contain valid values until the MediaElement raises the MediaOpened event. The following is a list of MediaElement properties that will not be valid until after MediaOpened is raised by that element.

·         Attributes

·         AudioStreamCount

·         AudioStreamIndex

·         BufferingProgress

·         DownloadProgress (MediaElement)

·         Markers

·         NaturalDuration

·         NaturalVideoHeight

·         NaturalVideoWidth

In addition, the Position property will be null before the MediaOpened event. However, Position is different from the other properties in the second group because it is settable. You should not attempt to set Position until MediaOpened is raised. You also should not set an initial Position value in XAML.

 

MediaFailed

This event can occur in the following conditions:

·         File not found.

·         Invalid (unrecognized or unsupported) media format.

·         Unknown media error during playback.

You may see this event fires when you debug your Silverlight application in visual studio. The URL of the video is valid. You can open the video in Windows Media Player.

To solve this issue, the first thing you need to check is whether the URL is a cross-scheme one. Common cause is, the URL of the page is of FILE scheme (e.g. “C:\\...”) and the URL of the video is of HTTP scheme (e.g. http://..). Vice versa.

For more details about this restriction, please check out:

http://msdn.microsoft.com/en-us/library/cc189008(VS.95).aspx

 

CurrentStateChanged

This event fires when the value of the CurrentState property changes. In the sample we catch some state change events and display current state on StatusTextBlock.

 

How to handle MediaElement in full-screen mode

 

A common requirement regarding MediaElement in full-screen mode is, we expect only the MediaElement to appear in the full-screen mode. This sample demonstrates how to do that. Basically the idea is to enlarge width/height of MediaElement and hide other controls when Silverlight is in full-screen mode. You can prepare another set of UIs to control MediaElement in full-screen mode.

The code regarding full-screen mode can also be found in our samples CSSL3FullScreen/ VBSL3FullScreen.
